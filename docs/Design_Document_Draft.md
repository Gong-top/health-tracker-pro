# 《个人健康管理与习惯追踪应用》设计文档

## 1. 选题说明及需求介绍
本项目旨在通过数字化的方式帮助用户管理个人健康与日常习惯。
- **痛点**: 现代人生活节奏快，难以量化健康数据。
- **功能目标**: 记录打卡、监控体重、生成健康月报、存储运动瞬间（图片附件）。

## 2. 系统功能模块划分
- **用户管理**: 注册、登录、个人资料修改。
- **习惯追踪**: 自定义习惯、打卡记录、**触发器自动统计**。
- **健康监控**: 体重、心率数据录入、**趋势图表展示**。
- **报告生成**: **存储过程**生成月度汇总报告、**视图**展示周报。
- **附件管理**: 打卡记录相关图片或文件的存储。

## 3. 数据库概念结构设计 (E-R 图设计思路)
- **核心实体**: 用户 (User), 习惯 (Habit), 健康记录 (HealthRecord), 运动项目 (Sports)。
- **关系**: 
    - 1个用户拥有多个习惯 (1:N)
    - 1个用户拥有多条打卡记录 (1:N)
    - 1条记录可关联多个附件 (1:N)
    - 用户习惯与习惯定义通过外键连接。

## 4. 数据库逻辑结构设计 (关系模式)
- `users(id, username, password, ...)` (1NF, 2NF, 3NF 规范化)
- `record(id, habit_id, date, status, ...)` (符合第三范式)
- `statistic(id, user_id, sta_date, ...)` (由触发器实时同步)

## 5. 数据库实施维护计划
- **安全性**: 对 `password` 进行加盐哈希存储；对敏感数据 `health_data` 设置行级访问权限（后端实现）。
- **完整性**: 使用 `on delete cascade` 级联删除；设置 `check` 约束（如心情指数 1-10）。
- **性能**: 在 `username`, `record_date` 上建立 `index` 索引，提升查询视图的效率。
- **维护**: 编写存储过程 `db_maintenance_plan` 定期清理陈旧数据。
